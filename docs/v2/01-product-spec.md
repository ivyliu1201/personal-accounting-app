# 產品規格 V2

## 1. 產品定位

本產品是個人記帳 Web App，以單一使用者帳號管理自己的收入與支出資料。

使用者透過 Google 登入，記錄收入與支出帳目，查看今日明細，並分析歷史現金流與類別分布。

## 2. MVP 範圍

V2 MVP 包含：

- 透過 Firebase Authentication 使用 Google 登入。
- 依使用者隔離帳目資料。
- 批次新增收入與支出帳目。
- 首頁今日明細列表。
- 首頁類別分析。
- 歷史查看日期查詢。
- 歷史查看收入與支出明細。
- 類別分布分析。
- 月度與年度現金流趨勢分析。
- 從歷史查看頁編輯與刪除單筆帳目。
- 預設類別。
- 在新增或編輯帳目時建立自訂類別。

## 3. 頁面

### 3.1 首頁

用途：

- 快速新增帳目。
- 快速查看本月資料。
- 快速查看今天的帳目。

主要區塊：

- 上方導覽列：產品名稱、首頁、歷史查看、登入或使用者資訊。
- 左側區塊：批次新增表單。
- 右上區塊：甜甜圈圖與類別摘要。
- 右下區塊：今日明細。

### 3.2 歷史查看

用途：

- 依日期查詢歷史帳目。
- 分析指定日期區間的類別分布。
- 查看年度每月現金流趨勢。
- 編輯與刪除帳目。

主要區塊：

- 上方導覽列。
- 左側區塊：日期查詢與明細列表。
- 右上區塊：甜甜圈圖與摘要模式切換。
- 右下區塊：年度現金流趨勢圖。

## 4. 登入與權限

### 4.1 登入

- 登入方式：Google 登入。
- 驗證服務：Firebase Authentication。
- 前端取得 Firebase ID token。
- 後端驗證 Firebase ID token。
- Firebase 使用者 ID 作為系統使用者 ID。

### 4.2 未登入狀態

- 可看到首頁版型。
- 不顯示真實使用者資料。
- 首頁操作不會變更資料。
- 點擊受保護操作時提示使用者登入。
- 不可進入歷史查看頁。
- 上方導覽列顯示登入操作。

### 4.3 已登入狀態

- 登入後回到首頁。
- 首頁切換為可操作狀態。
- 上方導覽列顯示使用者名稱或 email，以及登出操作。
- 使用者僅能存取自己的資料。

## 5. 帳目欄位

欄位：

- 類型：必填。值為收入或支出。
- 日期：必填。
- 金額：必填。
- 類別：必填。
- 備註：選填。
- 建立時間：系統產生，不可編輯。

驗證：

- 金額必須大於 0。
- 日期可選今天、過去日期或未來日期。
- 類別選項依類型切換。
- 類別可為預設類別或自訂類別。

排序定義：

- 帳目日期：使用者輸入的日期。
- 建立時間：帳目建立時由系統產生的時間。
- 首頁今日明細只顯示今天資料，並依類別排序。
- 歷史查看明細依帳目日期由新到舊排序，再依建立時間由新到舊排序。

日期定義：

- 今天使用系統目前的應用日期基準。
- 本月指本月第一天到今天。
- 歷史查詢與圖表使用帳目日期，除非該區塊另有明確說明。

## 6. 批次新增

- 預設顯示一列，類型預設為支出，日期預設為今天。
- 使用者可以新增列。
- 每列必填：類型、日期、金額、類別。
- 備註選填。
- 類別依類型切換。
- 可透過類別欄位建立自訂類別。
- 若任一列未完成，整批不可送出。
- 若送出失敗，整批視為失敗。
- 成功後：
  - 顯示成功訊息。
  - 清空表單。
  - 更新首頁甜甜圈圖、類別摘要、今日明細與類別選項。
- 失敗後：
  - 顯示失敗原因。
  - 保留已輸入資料供使用者修正。

## 7. 編輯與刪除

### 7.1 適用範圍

- 編輯與刪除僅提供於歷史查看明細列表。
- 首頁今日明細僅供查看。

### 7.2 編輯

- 由歷史查看明細列表的編輯按鈕觸發。
- 開啟彈窗。
- 可編輯欄位：
  - 類型。
  - 日期。
  - 金額。
  - 類別。
  - 備註。
- 不可編輯欄位：
  - 建立時間。
- 取消會關閉彈窗且不變更資料。
- 成功後：
  - 顯示成功訊息。
  - 保留目前頁碼、每頁筆數與查詢條件。
  - 更新目前頁面資料與相關圖表。
- 失敗後：
  - 顯示失敗原因。
  - 保留彈窗內容。
  - 不更新畫面資料。

### 7.3 刪除

- 由歷史查看明細列表的刪除按鈕觸發。
- 刪除前需顯示確認彈窗。
- 確認彈窗顯示日期、類別與金額。
- 取消會關閉彈窗且不刪除。
- 成功後：
  - 顯示成功訊息。
  - 更新目前列表與相關圖表。
  - 若目前歷史查看頁刪除後變成空頁，回到上一頁。
  - 若沒有上一頁資料，顯示空狀態。
- 失敗後：
  - 顯示失敗原因。
  - 不更新畫面資料。

## 8. 首頁需求

### 8.1 批次新增表單

首頁支援第 6 節定義的批次新增。

### 8.2 甜甜圈圖與類別摘要

模式：

- 支出類別。
- 收入類別。
- 總現金流。

規則：

- 預設模式為支出類別。
- 日期範圍固定為本月第一天到今天。
- 支出與收入模式顯示類別分布。
- 總現金流模式顯示收入與支出分布。
- 總現金流模式中心值顯示淨現金流。
- 類別摘要列表預設收合，使用者點開後顯示。
- 類別摘要列表與甜甜圈圖使用相同資料來源與模式。
- 類別摘要列表欄位：
  - 類別。
  - 金額。
  - 占比。
- 無資料時顯示空狀態。

### 8.3 今日明細

- 預設顯示 5 筆。
- 可切換顯示 10 或 15 筆。
- 只顯示帳目日期為今天的資料。
- 依類別排序，同類別內依建立時間由新到舊排序。
- 欄位：
  - 日期。
  - 類型。
  - 類別。
  - 金額。
  - 備註。
  - 建立時間。
- 備註過長時截斷。
- 不提供編輯或刪除。
- 「查看全部」進入歷史查看頁，不強制重設歷史查詢條件。

## 9. 歷史查看需求

### 9.1 日期查詢

- 支援日期區間查詢。
- 單日查詢以相同開始日期與結束日期表示。
- 預設區間為本月第一天到今天。
- 日期條件共用於歷史明細列表與歷史甜甜圈圖。
- 年度現金流趨勢年份依查詢開始日期決定。

### 9.2 明細類型與圖表類型

- 左側明細類型與右側甜甜圈圖模式可獨立控制。
- 變更日期時更新：
  - 左側明細。
  - 右側甜甜圈圖。
  - 年度現金流趨勢年份。
- 變更明細類型時，只更新明細列表。
- 變更圖表類型時，只更新甜甜圈圖顯示。

### 9.3 明細列表

- 預設每頁 10 筆。
- 每頁筆數選項：10、15、20。
- 分頁方式：上一頁與下一頁。
- 依帳目日期由新到舊排序，再依建立時間由新到舊排序。
- 欄位：
  - 日期。
  - 類型。
  - 類別。
  - 金額。
  - 備註。
- 可編輯與刪除。
- 無資料時顯示空狀態。

### 9.4 甜甜圈圖

模式：

- 支出類別。
- 收入類別。
- 總現金流。

規則：

- 使用目前歷史查看日期區間。
- 總現金流模式顯示收入與支出分布。
- 中心值顯示淨現金流。
- 無資料時顯示空狀態。
- 單日查詢仍顯示甜甜圈圖。

### 9.5 年度現金流趨勢

- 顯示某一年每月淨現金流與累積現金流。
- 年份依歷史查詢開始日期決定。
- 每月現金流 = 該月收入扣除支出。
- 累積現金流 = 從一月到該月的累積淨額。
- 該年度無資料時顯示空狀態。

## 10. 類別

預設支出類別：

- 飲食
- 交通
- 投資
- 繳費
- 自我成長
- 社交
- 治裝費
- 運動

預設收入類別：

- 投資
- 薪資

自訂類別規則：

- 可在新增或編輯帳目時建立。
- 歸屬於收入或支出。
- 歸屬於目前登入使用者。
- 同一使用者、同一類型、同一類別名稱不可重複。
- 收入與支出可以有相同類別名稱，因為兩者屬於不同類型範圍。
- V2 不提供獨立類別管理頁。
- V2 不支援自訂類別重新命名或刪除。

## 11. 空狀態與錯誤狀態

- 未登入時，受保護操作顯示登入提示。
- 查詢無資料時，相關列表或圖表顯示空狀態。
- 新增失敗時保留表單資料。
- 編輯失敗時保留彈窗資料。
- 刪除失敗時保留目前資料。
- loading 狀態應保持輕量，並靠近受影響區塊。

## 12. 非本階段範圍

除非先更新本文件，否則不得新增：

- 預算管理。
- 週期性帳目。
- 匯入或匯出。
- 多幣別。
- 共享帳本。
- 群組帳本。
- 發票掃描。
- OCR。
- AI 財務分析。
- 通知提醒。
- 遊戲化功能。
- 手機 App。
- 獨立類別管理頁。
